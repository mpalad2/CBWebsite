Platform Integrity

Welcome to the Platform Integrity Team Homepage!
The Platform Integrity Team ensures the wellbeing of CareerBuilder's internal products and provides security for CB's organizations on GitHub.
The PI team consists of five members: William Boatin, Ethan Latimer, Byron Long, Russell Baker, and Charles Kiser.

  • GitAudit It is an application that monitors the members of our GitHub organizations. Whether the member is allowed to stay or not is determined by some expandable policies: It is required that you allow us to view your private, validated emails. It is required that you must activate two factor authentication. It is required that your GitHub profile has your CareerBuilder email address. It is required that you are an active employee. Click here for more information and to check your compliance with these policies.

  • Two Factor Authentication To learn more about 2FA (two factor authentication), click here.

  • Product Status This website provides a way to glance at how well CareerBuilder is doing with its performance on its products. We show the uptime, and alert when we go below threshold. We also show the duration of common paths in each product (e.g login on the US website). Click here to view the page.

  • Datastore Datastore is a tracking tool used to track the activity of recruiters to help satisfy government audits. The tool collects data, which can used to generate a report for any given time frame and account. To generate a report, click here.

  • OFCCP The government program that runs Datastore is called OFCCP. Click here for more information about OFCCP.

  • Source Control We manage the source control systems for most of CareerBuilder. For the TFS for the matrix code, click here.

  • Github To access the GitHub for newer off-matrix code for source control, click here.

  • Team Portal The products are test environment, the automated test environment, test deployment, and production deployment. We are responsible for deploying code hourly to the test environment efficiently, for the health of the test environment, and for driving the tests developers write via our automated test environment that runs any valid unit and browser tests and reports the results to developers. You can view a summary and details of each test run on the team portal page by clicking here.

  • Team Portal Info This page shows you at a glance which snapshot of code was deployed and tested successfully, which set of code is the high water mark for what is currently in production, which snapshot of code failed deployment or had failing tests, the links to details about what code or flag changesets are in each snapshot of code, the links to details about which tests failed and why, and the data on which tests are problematic.

  • Production Deployment We are responsible for deploying the production environment every business day. To view the details of production deployments, click here.

  • Test Deployment We are responsible for deploying to the test environment hourly. To view test deployments, click here.

  • Filler This is a filler tile that does absolutely nothing.
